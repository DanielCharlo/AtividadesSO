Atividades Teoricas

1. Codigo aberto vs Codigo Fechado

O codigo aberto é aquele aonde o usuario pode vizualizar, modificar, distribuir e colaborar no codigo-fonte do sistema. Exemplos: FireFox, linux, android.

Vantagens: gratuito ou custo muito baixo, mais opções de personalização do sistema

Desvantagens: Dificil de utilizar (exige mais conhecimento tecnico), Pouco suporte oficial, dependendo da comunidade

O codigo fechado é aquele aonde o código-fonte do sistema é exclusivamente da empresa responsavel, fazendo com que os usuarios nao consigam vizualizar ou modificar. Exemplos: Windows, MacOS, iOS

Vantagens: Suporte tecnico oficial sem depender da comunidade, geralmente é mais facil de utilizar

Desvantagens: Custo elevado, geralmente as licenças são pagas, e pode ser menos customizavel ja que não se tem acesso ao código-fonte

2. Comparação de instalação

Na instalação de um sistema de código fechado, como o Windows, o usuário precisa comprar uma licença e inserir uma chave de ativação. O processo é simples e padronizado, com poucas opções de personalização, e pode exigir a instalação manual de alguns drivers após a conclusão

Já em um sistema de código aberto, como o Ubuntu, a instalação é gratuita e sem necessidade de chave. O usuário tem mais opções de personalização logo no início, como particionar o disco e instalar softwares adicionais. A maioria dos drivers é detectada automaticamente, mas pode ser necessário instalar alguns, como os de placas de vídeo

3. Modo de texto vs Interface Grafica

O modo de texto é utilizado principalmente por pessoas experientes, que têm conhecimento técnico ou que desejam ser mais precisas. Geralmente, este modo precisa de pouco hardware. Já a interface gráfica é mais intuitiva, o que facilita para pessoas que têm pouco conhecimento técnico, tornando o sistema mais acessível e visual. Porém, necessita de mais hardware, deixando as operações mais lentas, repetitivas ou complexas. Os cenários são: O modo texto é ideal para administradores de sistemas, desenvolvedores e usuários que precisam de automação, controle total e desempenho otimizado, como em servidores Linux. Já a interface gráfica é mais indicada para o uso cotidiano de pessoas comuns, em atividades como navegação na internet, edição de documentos ou jogos.

4. Analise Estrutura de Diretórios

Windows: A estrutura de diretórios do Windows é organizada principalmente a partir de unidades de disco, geralmente identificadas por letras. Exemplo: C: Diretórios: C:\Windows – Possui os arquivos do sistema operacional, bibliotecas, drivers e componentes necessários para o funcionamento do sistema. C:\Program Files – Lugar onde são instalados os aplicativos. C:\Users – Armazena os perfis dos usuários do sistema, contendo documentos, configurações pessoais e pastas como Downloads, Documents e Desktop.

A estrutura de diretórios do Linux segue o padrão FHS, onde tudo se organiza a partir do diretório raiz: / Exemplo: /bin : Contém os executáveis essenciais para o funcionamento do sistema, usados tanto pelo administrador quanto pelos usuários comuns. Exemplos: ls (listar arquivos), cp (copiar arquivos), mv (mover arquivos). /etc : Armazena arquivos de configuração do sistema e de diversos serviços e programas. Exemplos: /etc/passwd (informações de contas de usuário), /etc/hostname (nome do host do sistema). /home : Diretório onde ficam os arquivos pessoais dos usuários, semelhante ao C:\Users no Windows. Exemplo: /home/charlo contém documentos, configurações e arquivos do usuário "charlo".

5. Modelos de Usuários e Permissões

O gerenciamento de usuários e permissões serve para controlar quem pode acessar ou modificar arquivos e pastas, garantindo segurança no sistema.

O Windows Usa um sistema chamado ACL (Lista de Controle de Acesso), onde é possível definir permissões bem detalhadas para cada usuário ou grupo, como ler, escrever ou executar. A configuração é feita de forma fácil pela interface gráfica. Muito usado em empresas com Active Directory para gerenciar vários usuários.

Linux Usa um modelo mais simples, baseado em três tipos de usuários: dono, grupo e outros. As permissões são: r (ler), w (escrever) e x (executar). É configurado principalmente pela linha de comando, mas é rápido e eficiente. Ideal para servidores e quem quer mais controle e automação.
